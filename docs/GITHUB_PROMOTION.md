## 7 天推广计划

第 1 天：

- 完善 GitHub About、Topics、README 截图。
- 创建 v1.0.0 Release。
- 自己从头按 README 跑一遍，确保新人能启动。

第 2 天：

- 发一篇 CSDN 或博客园文章。
- 标题带上 Spring Boot、MyBatis-Plus、课程设计、个人财务管理系统。

第 3 天：

- 在掘金或知乎发项目介绍。
- 重点展示页面截图、功能列表、技术栈和运行步骤。

第 4 天：

- 发到 Java 学习群、课程设计交流群、朋友圈或 QQ 空间。
- 文案不要刷屏，重点说明“适合课程设计参考”。

第 5 天：

- 增加一个小功能或修复一个问题，提交一次有意义的 commit。
- 在 README 添加 Roadmap 或扩展方向。

第 6 天：

- 录制 1 分钟运行演示视频，发布到 B 站动态、抖音或视频号。
- 视频内容：启动项目、登录、添加流水、查看报表。

第 7 天：

- 复盘 GitHub Insights。
- 看哪些来源带来访问，再继续投放到有效平台。

## 不建议做的事

- 不要买 Star。
- 不要用脚本刷访问量。
- 不要到别人项目下面刷广告。
- 不要夸大功能，README 写真实能力即可。
- 不要把账号密码、数据库密码、Token 提交到仓库。

真实 Star 来得慢，但质量更高。开源项目最重要的是：能跑、好看、文档清楚、持续更新。
